# 系统架构概览

TradeMaster Web Interface采用现代化的微服务架构设计，结合前后端分离和容器化部署，提供高性能、高可用、易扩展的量化交易平台。

# 核心数据模型

  用户表 (users):
    - id: 主键
    - username: 用户名
    - email: 邮箱
    - password_hash: 密码哈希
    - role: 角色
    - created_at: 创建时间
    - updated_at: 更新时间
    
  会话表 (user_sessions):
    - id: 主键
    - user_id: 用户ID
    - session_token: 会话令牌
    - ip_address: IP地址
    - user_agent: 用户代理
    - expires_at: 过期时间
    
  API密钥表 (api_keys):
    - id: 主键
    - user_id: 用户ID
    - key_name: 密钥名称
    - key_hash: 密钥哈希
    - permissions: 权限列表
    - is_active: 是否激活

策略域 (Strategy Domain):
  策略表 (strategies):
    - id: 主键
    - user_id: 用户ID
    - name: 策略名称
    - type: 策略类型
    - config: 策略配置
    - status: 策略状态
    - version: 版本号
    
  策略版本表 (strategy_versions):
    - id: 主键

    - config: 配置快照
    - created_at: 创建时间

  策略执行表 (strategy_executions):
    - id: 主键

    - start_time: 开始时间
    - end_time: 结束时间
    - status: 执行状态
    - result: 执行结果

数据域 (Data Domain):
  数据集表 (datasets):
    - id: 主键
    - user_id: 用户ID
    - name: 数据集名称
    - type: 数据类型
    - source: 数据来源
    - file_path: 文件路径
    - metadata: 元数据
    
  市场数据表 (market_data):
    - id: 主键
    - symbol: 交易标的
    - timestamp: 时间戳
    - open: 开盘价
    - high: 最高价
    - low: 最低价
