## 📊 数据类型和格式

TradeMaster支持多种类型的金融数据，满足不同策略的需求：

### 基础价格数据 (OHLCV)

这是最常用的价格数据格式，包含开高低收和成交量信息：

```csv
# 标准OHLCV格式
date,open,high,low,close,volume
2024-01-01 00:00:00,42000.5,43500.0,41800.0,43200.0,1250.5
2024-01-01 01:00:00,43200.0,44100.0,42900.0,43800.0,1100.2
2024-01-01 02:00:00,43800.0,44500.0,43600.0,44200.0,980.8
```

**字段说明**：
- `date`: 时间戳，支持多种格式
- `open`: 开盘价
- `high`: 最高价
- `low`: 最低价  
- `close`: 收盘价
- `volume`: 成交量（可选）

**支持的时间格式**：
```
日期时间格式示例:
├── ISO格式: 2024-01-01T09:30:00Z
├── 标准格式: 2024-01-01 09:30:00
├── 日期格式: 2024-01-01
├── 时间戳: 1704067800
└── Excel格式: 2024/1/1 9:30:00
```

### 高频交易数据 (Tick Data)

用于高频交易策略的逐笔交易数据：

```csv
# Tick数据格式
timestamp,price,volume,side
2024-01-01 09:30:00.123,43200.5,0.1,buy
2024-01-01 09:30:00.156,43201.0,0.05,sell
2024-01-01 09:30:00.189,43200.0,0.2,buy
```

**字段说明**：
- `timestamp`: 精确到毫秒的时间戳
- `price`: 成交价格
- `volume`: 成交数量
- `side`: 买卖方向 (buy/sell)

### 订单簿数据 (Order Book)

市场深度数据，显示买卖盘情况：

```csv
# 订单簿数据格式
timestamp,bid_price_1,bid_size_1,ask_price_1,ask_size_1,bid_price_2,bid_size_2,ask_price_2,ask_size_2
2024-01-01 09:30:00,43200.0,1.5,43201.0,2.1,43199.5,0.8,43202.0,1.2
```

### 基本面数据

企业财务和宏观经济数据：

```csv
# 基本面数据格式
date,symbol,revenue,net_income,total_assets,pe_ratio,pb_ratio
2024-Q1,AAPL,90750000000,23630000000,365725000000,28.5,6.2
```

### 新闻和情感数据

市场情绪和新闻数据：

```csv
# 情感数据格式
date,symbol,sentiment_score,news_count,positive_ratio,negative_ratio
2024-01-01,BTC,0.65,156,0.42,0.23
```

#### 2. API接口上传

```python
# Python示例代码
import requests
import pandas as pd

# 准备数据
data = pd.read_csv('btc_data.csv')

# 上传API
url = "http://localhost:8000/api/v1/data/upload"
headers = {
    "Authorization": "Bearer YOUR_TOKEN",
    "Content-Type": "application/json"
}

payload = {
    "dataset_name": "BTC_Daily_2024",
    "data_type": "ohlcv",
    "symbol": "BTC-USDT",
    "frequency": "1d",
    "timezone": "UTC",
    "data": data.to_dict('records')
}

response = requests.post(url, json=payload, headers=headers)
print(response.json())
```

## 🔧 数据预处理

数据预处理是提高数据质量和模型性能的关键步骤：

### 数据清洗

#### 缺失值处理

```python
# 缺失值处理方法
处理方式选择:

1️⃣ 删除法
├── 删除含缺失值的行
├── 适用: 缺失比例 < 5%
├── 优点: 简单直接
└── 缺点: 可能丢失重要信息

2️⃣ 前向填充 (forward fill)
├── 用前一个值填充
├── 适用: 价格数据
├── 优点: 保持连续性
└── 缺点: 可能引入偏差

3️⃣ 后向填充 (backward fill)
├── 用后一个值填充
├── 适用: 预处理阶段
├── 优点: 避免前视偏差
└── 缺点: 实时数据无法使用

4️⃣ 插值法
├── 线性插值/样条插值
├── 适用: 连续型数据
├── 优点: 保持数据趋势
└── 缺点: 计算复杂

5️⃣ 统计填充
├── 均值/中位数/众数
├── 适用: 非时间序列数据
├── 优点: 统计性质稳定
└── 缺点: 可能平滑异常值
```

#### 异常值检测和处理

```yaml
# 异常值检测方法
异常值检测:
  
统计方法:
  - 3-sigma规则: |x - μ| > 3σ
  - IQR方法: Q1 - 1.5*IQR < x < Q3 + 1.5*IQR
  - 百分位数: 1%和99%分位数外的值
  
时间序列方法:
  - 价格跳跃检测: |return| > threshold
  - 成交量异常: volume > μ + 5σ
  - 波动率异常: volatility > historical_max * 2
  
机器学习方法:
  - Isolation Forest
  - Local Outlier Factor
  - One-Class SVM

异常值处理:
  替换策略:
    - 上下限截断 (winsorization)
    - 用邻近值替换
    - 用统计值替换
  
  标记策略:
    - 保留原值但标记异常
    - 后续分析时特别处理
    - 作为特征加入模型
```

#### 自定义特征

```python
# 自定义特征示例
class CustomFeatures:
    def __init__(self, data):
        self.data = data
    
    def price_features(self):
        """价格相关特征"""
        features = {}
        
        # 收益率特征
        features['returns_1d'] = self.data['close'].pct_change()
        features['returns_5d'] = self.data['close'].pct_change(5)
        features['returns_20d'] = self.data['close'].pct_change(20)
        
        # 价格相对位置
        features['price_position_20d'] = (
            self.data['close'] - self.data['close'].rolling(20).min()
        ) / (
            self.data['close'].rolling(20).max() - self.data['close'].rolling(20).min()
        )
        
        # 高低价差
        features['high_low_ratio'] = self.data['high'] / self.data['low']
        features['close_to_high'] = self.data['close'] / self.data['high']
        features['close_to_low'] = self.data['close'] / self.data['low']
        
        return features
    
    def volume_features(self):
        """成交量相关特征"""
        features = {}
        
        # 成交量变化
        features['volume_change'] = self.data['volume'].pct_change()
        features['volume_ma_ratio'] = (
            self.data['volume'] / self.data['volume'].rolling(20).mean()
        )
        
        # 价量关系
        features['price_volume_corr'] = (
            self.data['close'].rolling(20).corr(self.data['volume'])
        )
        
        return features
    
    def volatility_features(self):
        """波动率特征"""
        features = {}
        
        # 历史波动率
        returns = self.data['close'].pct_change()
        features['volatility_5d'] = returns.rolling(5).std() * np.sqrt(252)
        features['volatility_20d'] = returns.rolling(20).std() * np.sqrt(252)
        
        # 实现波动率
        features['realized_vol'] = np.sqrt(
            ((self.data['high'] / self.data['close']).rolling(20).var() +
             (self.data['low'] / self.data['close']).rolling(20).var()) / 2
        )
        
        return features
```

### 数据标准化和归一化

```python
# 数据标准化方法
标准化选择:

1️⃣ Z-score标准化
公式: z = (x - μ) / σ
适用: 数据服从正态分布
特点: 均值为0，标准差为1

2️⃣ Min-Max标准化  
公式: x' = (x - min) / (max - min)
适用: 有明确最大最小值
特点: 数据范围[0,1]

3️⃣ 分位数标准化
公式: x' = rank(x) / n
适用: 非正态分布数据
特点: 转换为均匀分布

4️⃣ 滚动标准化
公式: z_t = (x_t - μ_t-n:t) / σ_t-n:t
适用: 时间序列数据
特点: 避免前视偏差

5️⃣ 对数变换
公式: x' = log(x)
适用: 价格数据
特点: 转换为收益率
```
