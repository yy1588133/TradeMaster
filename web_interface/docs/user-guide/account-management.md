  用户名: "trademaster_user"

    - 登录凭证
    - 接收系统通知
    - 密码重置使用
    - 修改需要验证

  手机号码: "+86 138****8888"
    - 安全验证使用

    - 支持国际号码

## 🔒 安全设置

### 密码管理

账户安全的第一道防线：

#### 密码强度要求

```yaml
# 密码安全规范
密码要求:

基本要求:
  最小长度: 8个字符
  最大长度: 128个字符
  字符类型: 至少包含3种类型
    - 小写字母 (a-z)
    - 大写字母 (A-Z)  
    - 数字 (0-9)
    - 特殊字符 (!@#$%^&*)

强度评级:
  弱密码 (0-40分):
    - 过于简单
    - 不推荐使用
    - 系统强制升级
    
  中等密码 (41-70分):
    - 基本安全
    - 可以使用
    - 建议加强
    
  强密码 (71-90分):
    - 安全可靠
    - 推荐使用
    - 定期更换
    
  超强密码 (91-100分):
    - 极其安全
    - 最佳选择
    - 长期有效

禁用密码:
  - 与用户名相同
  - 常见弱密码 (123456, password等)
  - 近期使用过的密码
  - 个人信息相关密码
```

#### 密码修改流程

```
🔐 密码修改步骤：

1️⃣ 身份验证
├── 输入当前密码
├── 验证登录状态
├── 检查权限级别
└── 确认用户身份

2️⃣ 新密码设置
├── 输入新密码
├── 实时强度检测
├── 确认新密码
└── 检查密码规则

3️⃣ 安全确认
├── 邮箱验证码
├── 手机验证码 (如已绑定)
├── 安全问题验证
└── 最终确认修改

4️⃣ 完成修改
├── 系统更新密码
├── 发送确认邮件
├── 记录安全日志
└── 强制重新登录
```

### 两步验证 (2FA)

增强账户安全的重要功能：

#### 2FA设置选项

```yaml
# 两步验证配置
2FA选项:

验证器应用:
  推荐应用:
    - Google Authenticator
    - Microsoft Authenticator
    - Authy
    - 1Password
    
  设置流程:
    1. 扫描QR码
    2. 输入6位验证码
    3. 保存备份密钥
    4. 完成绑定
    
短信验证:
  支持功能:
    - 登录验证
    - 敏感操作确认
    - 异常登录提醒
    
  注意事项:
    - 需要绑定手机号
    - 可能产生短信费用
    - 网络延迟影响
    
邮箱验证:
  适用场景:
    - 备用验证方式
    - 海外用户选择
    - 批量操作确认
    
  特点:
    - 免费使用
    - 全球可用
    - 支持富文本

备份码:
  重要性: 紧急情况下的访问方式
  数量: 10个一次性使用备份码
  保存建议:
    - 打印并安全保存
    - 不要截图存储
    - 使用后及时更新
    - 定期检查有效性
```

#### 2FA使用流程

```
🛡️ 两步验证使用：

正常登录流程:
1️⃣ 输入用户名和密码
2️⃣ 系统提示需要2FA验证
3️⃣ 打开验证器应用
4️⃣ 输入6位动态码
5️⃣ 完成登录验证

紧急访问流程:
1️⃣ 无法获取动态码
2️⃣ 选择"使用备份码"
3️⃣ 输入一次性备份码  
4️⃣ 成功登录后及时修复2FA

2FA恢复流程:
1️⃣ 联系客服验证身份
2️⃣ 提供身份证明材料
3️⃣ 通过人工审核
4️⃣ 重置2FA设置
```

## 🔑 API密钥管理

### API密钥概述

为程序化交易和系统集成而设计：

#### API密钥类型

```yaml
# API密钥分类
密钥类型:

按权限级别:
  只读密钥 (Read-Only):
    - 查看账户信息
    - 获取策略数据
    - 查看交易历史
    - 访问分析报告
    
  交易密钥 (Trading):
    - 只读权限 + 
    - 创建策略
    - 执行交易
    - 修改订单
    
  管理密钥 (Management):
    - 交易权限 +
    - 用户管理
    - 系统配置
    - 敏感操作

按使用场景:
  开发测试 (Development):
    - 用于开发环境
    - 有访问限制
    - 低速率限制
    - 测试数据访问
    
  生产环境 (Production):
    - 用于正式环境
    - 完整功能访问
    - 正常速率限制
    - 实时数据访问
    
  第三方集成 (Integration):
    - 用于第三方应用
    - 受限权限范围
    - 严格速率控制
    - 审计日志记录
```

### 创建API密钥

#### 创建流程

```
🔑 API密钥创建步骤：

1️⃣ 进入API管理页面
├── 导航至"账户设置" → "API密钥"
├── 点击"创建新密钥"按钮
└── 选择密钥创建模式

2️⃣ 配置密钥参数
├── 密钥名称: "量化交易机器人"
├── 权限级别: "交易密钥"
├── IP白名单: "192.168.1.100, 10.0.0.1"
├── 速率限制: "1000请求/分钟"
├── 有效期: "1年"
└── 使用描述: "用于自动化交易系统"

3️⃣ 安全验证
├── 输入账户密码
├── 2FA动态验证码
├── 邮箱确认链接
└── 最终确认创建

4️⃣ 密钥生成
├── 系统生成API密钥对
├── 显示API Key和Secret
├── 提供安全保存提示
└── 记录创建日志

⚠️ 重要提醒:
- Secret只显示一次，请立即安全保存
- 丢失Secret需要重新创建密钥
- 定期轮换密钥确保安全
```

#### 密钥配置详解

```yaml
# API密钥配置参数
配置选项:

基本信息:
  密钥名称:
    - 便于识别的描述性名称
    - 建议包含用途说明
    - 支持中英文，最长50字符
    
  密钥描述:
    - 详细说明使用场景
    - 记录创建目的
    - 便于后期管理

权限设置:
  功能权限:
    - 账户查询: 查看账户信息
    - 数据访问: 获取市场和策略数据
    - 策略管理: 创建和修改策略
    - 交易执行: 下单和撤单操作
    - 用户管理: 管理子账户和权限
    
  数据权限:
    - 实时数据: 获取实时市场数据
    - 历史数据: 查询历史数据
    - 个人数据: 访问用户个人信息
    - 敏感数据: 访问财务和交易数据

安全设置:
  IP白名单:
    - 限制访问IP地址
    - 支持IP段配置
    - 动态IP支持
    - 最多50个IP地址
    
  速率限制:
    - 每分钟请求次数限制
    - 防止恶意调用
    - 不同权限级别不同限制
    - 可根据需要调整
    
  有效期设置:
    - 1个月: 适合临时使用
    - 3个月: 适合短期项目
    - 1年: 适合长期使用
    - 永久: 仅限特殊情况
```
